# Offer Builder — конструктор оффера (уравнение ценности Хормози)

Назначение: помочь ЛЮБОМУ пользователю собрать сильный оффер и сохранить его как **профиль оффера** — источник правды, который читают все остальные скиллы системы (reels, carousel, post, stories, article, сайт, квиз). Без оффера контент бьёт мимо.

Это скилл-конструктор, а не шаблон. Не подставляй чужой оффер. Веди пользователя через его собственный.

## Базовый принцип — уравнение ценности

```
              Результат-мечта  ×  Вероятность его достижения
Ценность  =  ──────────────────────────────────────────
                 Время до результата  ×  Усилия и жертвы
```

Сильный оффер = максимум в числителе, минимум в знаменателе.

## Шаги
1. **Собрать вводные** — ниша/продукт, клиент, результат-мечта, боль, цена/формат. Недостающее спрашивай через AskUserQuestion по одному блоку.
2. **4 рычага**: результат-мечта (↑, конкретный измеримый итог), вероятность (↑, кейсы/гарантия), время (↓, быстрый первый результат), усилия (↓, сделать за клиента).
3. **Оффер-стек**: ядро + бонусы, каждый закрывает боль/возражение; сумма ценности >> цена.
4. **Гарантия**: безусловная / условная по результату / анти-гарантия.
5. **Имя оффера**: 3–5 вариантов, называют результат.
6. **Продуктовая лестница**: бесплатный вход → основной → премиум.
7. **Сохранить профиль** в Google Drive (документ/вкладка «Оффер»); если коннектора нет — `profile/offer.md`.

## Шаблон профиля
```markdown
# ПРОФИЛЬ ОФФЕРА
Ниша: ...
Клиент (аватар в одну строку): ...
Результат-мечта: ...
Главная боль: ...
Оффер-стек: [ядро + бонусы]
Гарантия: ...
Имя оффера: ...
Продуктовая лестница: бесплатный → основной → премиум
Цена: ...
```

## Контроль качества
Результат конкретный? Ценность стека >> цена? Каждый бонус снимает возражение? Гарантия снижает риск? Имя называет результат? Если нет — доработай.
