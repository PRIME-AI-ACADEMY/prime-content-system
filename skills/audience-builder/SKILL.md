# Audience Builder — конструктор ДНК клиента

Назначение: собрать глубинный портрет аудитории и сохранить как **профиль аудитории** — источник правды для контент-скиллов. Лучше всего работает после offer-builder.

## Шаги
1. **Вводные**: продукт/результат, кто покупает чаще, в какой момент ищет. Спрашивай по одному блоку.
2. **Аватар (1–3)**: кто (портрет в 1–2 предл.), ситуация сейчас, желаемый результат.
3. **Боли, желания, страхи** — по 3–5, словами клиента.
4. **Лестница Ханта**: на каких ступенях основная масса (от «нет проблемы» до «готов купить») — от этого тип контента.
5. **Возражения**: топ-5 + контраргумент.
6. **Язык клиента**: реальные фразы для хуков и заголовков.
7. **Сохранить профиль** в Google Drive («Аудитория»); иначе `profile/audience.md`.

## Шаблон
```markdown
# ПРОФИЛЬ АУДИТОРИИ
Аватар 1: [портрет]
  Боли: ... / Желания: ... / Страхи: ... / Ступень: ...
Топ-возражения + ответы: ...
Язык клиента (фразы для хуков): ...
```

## Контроль качества
Аватар — конкретный человек? Боли словами клиента? Понятно, на какой ступени писать? Есть лексика для хуков?
