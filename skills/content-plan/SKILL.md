# Content Plan — оркестратор контент-плана

Назначение: превратить цель в **рабочую таблицу-календарь** и довести до готового контента. ГЛАВНОЕ: сначала план на утверждение, потом детальные промпты, потом таблица.

## Шаг 0 — фундамент (обязательно)
Прочитай профили; чего нет — запусти конструктор: `business-dna`, `audience-builder`, `hunt-ladder`, `offer-builder`, `tone-of-voice-builder`, `design-system-builder`. Без `business-dna` и `audience` план не строй.

### Шаг 0.5 — дизайн-система ПЕРЕД планом (обязательно)
Если профиля `design-system` нет — сначала запусти `design-system-builder`. Спроси: любимые **цвета** (или 2–3 палитры) и **шрифты**. Только потом план.

## Шаг 1 — цель и период
Цель в числах, срок, ключевое событие, сколько публикаций реалистично.

## Шаг 2 — стратегия периода
По лестнице Ханта: баланс прогрев (1–3) / продажа (4–5) / вовлечение, привязка к событию.

## Шаг 3 — ПРЕДВАРИТЕЛЬНЫЙ план В ЧАТ (сначала на утверждение — НЕ в таблицу)
Сначала покажи пользователю **предварительный план прямо в чате** — компактным списком/табличкой:
```
День/Дата · Формат · Тема · Идея/угол · Хук · Ступень Ханта · CTA
```
БЕЗ детальных промптов, БЕЗ записи в Google-таблицу. Каждая тема — сильная, неочевидная идея/угол (не «5 ошибок» вообще, а конкретный заход). Спроси: **«Утверждаешь план или что поменять?»** Итерируй, пока не одобрит. Только после «утверждаю» — дальше.

## Шаг 3.5 — ДЕТАЛЬНАЯ ПРОРАБОТКА промптов (после утверждения плана)
Для КАЖДОЙ утверждённой строки пропиши промпты — **максимально детальные, готовые к запуску без ручной правки**, с сильной идеей под тему. Промпты РАЗНЫЕ под формат (не один шаблон):
- **Карусель кодом:** текст КАЖДОГО слайда (reads exactly), раскладка, палитра/шрифты из `design-system`, какие фото из банка/пула, где скрим для читаемости (text-and-photo-rules).
- **Карусель/картинки Higgsfield:** визуальный промпт строго по `carousel/references/higgsfield-visual-standard.md` — каждый кадр своя сцена, реальные фото-референсы лица, единый образ, бренд-палитра, текст построчно «reads exactly: …» + «every word spelled correctly», 9:16/4:5.
- **Видео (Reels):** укажи движок — **Higgsfield (kling, start/end кадры)** или **Seedance** под задачу; аватар lip-sync или кино-сцены; полный сценарий по copy-engine; озвучка — клон голоса или качественный русский голос; текст на экран/субтитры; B-roll сцены разные.
- **Пост/сторис:** финальный текст по copy-engine.
Перед любым текстом/промптом: `Read ${CLAUDE_PLUGIN_ROOT}/skills/content-pipeline/references/copy-engine.md` и прогони аудит (нет «не X, а Y», нет скобок-рода). Каждая тема — одна сильная идея, а не общая вода. Цель: пользователь НЕ правит промпты вручную.

## Шаг 4 — перенос в КРАСИВУЮ РАБОЧУЮ таблицу
Только ТЕПЕРЬ запиши детально проработанные строки на лист «Контент-план». Статус = «План».
**ФИКСИРОВАННАЯ схема 14 колонок, БЕЗ эмодзи в заголовках** (одинакова для всех):
```
Дата | Формат | Тема | Хуки (3 варианта) | Ступень Ханта | CTA / ключ | Higgsfield-промпт / Дизайн | Voiceover / Текст слайдов | Медиа (из банка) | Источник для скриншота | Ссылка на медиа (Drive) | Статус | Postiz (запланировано) | Зашло / Не зашло
```
В «Higgsfield-промпт / Дизайн» и «Voiceover / Текст слайдов» клади те самые детальные промпты из Шага 3.5.
**Это РАБОЧАЯ таблица, которая пополняется:** неделя → +неделя → месяц, дописывай строки НА ТОМ ЖЕ листе (не новый лист каждый раз).
Оформление — **никогда чёрный текст на белом** (GOOGLESHEETS_FORMAT_CELL + UPDATE_SHEET_PROPERTIES + UPDATE_DIMENSION_PROPERTIES + SET_DATA_VALIDATION_RULE):
- Шапка: тёмный фирменный фон (pine #00311e), белый жирный текст по центру, закреплена (freeze row 1), без эмодзи.
- Тело: бренд-беж (#fef7e5), перенос по словам (WRAP), выравнивание сверху.
- Ширина: текстовые широкие (Тема ~220, Хуки ~300, Higgsfield-промпт и Voiceover ~340, CTA ~200), служебные узкие. Текст НЕ обрезается.
- Статус — выпадающий список: План · На утверждении · Утверждено · Запланировано.
Цвета — из `design-system`. Покажи ссылку на таблицу и спроси, что генерировать.

## Шаг 5 — производство (по утверждению)
Для выбранных строк вызывай `content-pipeline`/формат-скилл по уже готовым промптам. Медиа → Google Drive, ссылка → в строку, статус → «На утверждении». ОБЯЗАТЕЛЬНЫЙ QA визуала (кириллица/лицо/сцена) с перегенерацией брака.

## Шаг 6 — цикл утверждения
В таблице «Утверждено» / «Переделать». По «Утверждено» — автопостинг (статус → «Запланировано»).

## Контроль качества
- СНАЧАЛА предварительный план в чат на утверждение, ПОТОМ таблица?
- Промпты детальные, разные под формат (код-карусель ≠ Higgsfield ≠ Seedance-видео), готовы без ручной правки?
- 14 колонок без эмодзи, статус-дропдаун, шапка закреплена, текст не обрезан?
- Текст без «не X, а Y» и скобок-рода (copy-engine аудит)?
