# Content Plan — оркестратор контент-плана

Назначение: цель → **рабочая таблица-календарь** → готовый контент. Минимум ручных утверждений: только (1) план и (2) образец из 2 постов. Дальше — автоматом.

## Шаг 0 — фундамент (обязательно)
Прочитай профили; чего нет — запусти конструктор: `business-dna`, `audience-builder`, `hunt-ladder`, `offer-builder`, `tone-of-voice-builder`, `design-system-builder`. Без `business-dna` и `audience` план не строй.

### Шаг 0.5 — дизайн-система ПЕРЕД планом (обязательно)
Если профиля `design-system` нет — сначала запусти `design-system-builder`. Спроси: любимые **цвета** (или 2–3 палитры) и **шрифты**. Только потом план.

## Шаг 1 — цель и период
Цель в числах, срок, ключевое событие, сколько публикаций реалистично.

## Шаг 2 — стратегия периода
По лестнице Ханта: баланс прогрев (1–3) / продажа (4–5) / вовлечение, привязка к событию.

## Шаг 3 — ПРЕДВАРИТЕЛЬНЫЙ план В ЧАТ (утверждение № 1 — НЕ в таблицу)
Покажи план на неделю прямо в чате — компактно:
```
День/Дата · Формат · Тема · Идея/угол · Хук · Ступень Ханта · CTA
```
Без детальных промптов, без записи в таблицу. Каждая тема — сильная, конкретная идея. Спроси: «Утверждаешь план?» Итерируй до одобрения.

## Шаг 3.5 — ОБРАЗЕЦ из 2 постов (утверждение № 2, последнее)
Возьми из плана **2 поста РАЗНЫХ форматов (напр. 1 карусель + 1 reels)** и распиши их ПОЛНОСТЬЮ, детально — так, как пойдёт в таблицу (все колонки/промпты: хуки, Higgsfield-промпт, текст слайдов/сценарий, CTA, медиа из банка и т.д.). Промпты РАЗНЫе под формат (код-карусель: текст каждого слайда reads-exactly + палитра/фото; Higgsfield: по higgsfield-visual-standard; видео: движок Higgsfield-kling или Seedance + аватар/озвучка; текст по copy-engine). Покажи эти 2 образца в чате. Спроси: **«Стиль и качество ок? Так делаю остальные?»** Одна правка при необходимости. Это **ЕДИНСТВЕННЫЙ контроль качества промптов** — дальше без пошагового утверждения.
Перед любым текстом: `Read ${CLAUDE_PLUGIN_ROOT}/skills/content-pipeline/references/copy-engine.md` + аудит (нет «не X, а Y», нет скобок-рода).

## Шаг 4 — ОСТАЛЬНЫЕ АВТОМАТОМ + вся таблица (без утверждения каждого)
После одобрения образца — распиши ОСТАЛЬНЫЕ строки в ТОМ ЖЕ стиле/качестве и СРАЗУ запиши ВСЮ таблицу. Не спрашивай утверждение по каждому промпту.
Лист «Контент-план», статус = «План». **ФИКСИРОВАННАЯ схема 14 колонок, БЕЗ эмодзи:**
```
Дата | Формат | Тема | Хуки (3 варианта) | Ступень Ханта | CTA / ключ | Higgsfield-промпт / Дизайн | Voiceover / Текст слайдов | Медиа (из банка) | Источник для скриншота | Ссылка на медиа (Drive) | Статус | Postiz (запланировано) | Зашло / Не зашло
```
**РАБОЧАЯ таблица, пополняется:** неделя → +неделя → месяц НА ТОМ ЖЕ листе.
Оформление — **никогда чёрный текст на белом:** шапка тёмный фирменный фон + белый жирный текст + закреплена + без эмодзи; тело бренд-беж + перенос по словам; ширина колонок без обрезки (текстовые широкие); Статус — выпадающий список (План · На утверждении · Утверждено · Запланировано). Цвета — из `design-system`. Покажи ссылку на таблицу.

## Шаг 5 — производство (по утверждению)
Для выбранных строк вызывай `content-pipeline`/формат-скилл по готовым промптам. Медиа → Drive, ссылка → в строку, статус → «На утверждении». QA визуала (кириллица/лицо/сцена) с перегенерацией брака.

## Шаг 6 — цикл утверждения
В таблице «Утверждено» / «Переделать». По «Утверждено» — автопостинг (статус → «Запланировано»).

## Контроль качества
- Порядок: план в чат → образец 2 постов → (одобрение) → вся таблица автоматом. Только 2 утверждения.
- Промпты детальные, разные под формат, готовы без ручной правки?
- 14 колонок без эмодзи, статус-дропдаун, шапка закреплена, текст не обрезан?
- Текст без «не X, а Y» и скобок-рода?
