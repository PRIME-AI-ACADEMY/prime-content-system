# Content Plan — оркестратор контент-плана

Назначение: цель → **рабочая таблица-календарь** → готовый контент. Минимум ручных утверждений: только (1) план и (2) образец из 2 постов. Дальше — автоматом.

## Шаг 0 — фундамент (обязательно)
Прочитай профили; чего нет — запусти конструктор: `business-dna`, `audience-builder`, `hunt-ladder`, `offer-builder`, `tone-of-voice-builder`, `design-system-builder`. Без `business-dna` и `audience` план не строй.

### Шаг 0.5 — дизайн-система ПЕРЕД планом (обязательно)
Если профиля `design-system` нет — сначала запусти `design-system-builder`. Спроси: любимые **цвета** (или 2–3 палитры) и **шрифты**. Только потом план.

## Шаг 1 — цель, период, МИКС форматов
Цель в числах, срок, ключевое событие.
- **Период гибкий: неделя / 2 недели / месяц** (спроси; по умолчанию неделя; месяц = 4 недельных блока).
- **Микс на неделю по умолчанию: 3 карусели + 3 reels** (6 постов; подстрой под ресурс). Карусели — РАЗНЫХ визуальных типов (НЕ однотипные).

## Шаг 1.5 — ТИПЫ КАРУСЕЛЕЙ (назначь каждой — обязательно)
Каждой карусели присвой **визуальный тип** и укажи его в колонке «Формат»:
- **Тип A · Текст-на-фоне** — код, брендовый фон + типографика. Быстро, чисто; для списков/чек-листов.
- **Тип B · С фото/аватаром** — Higgsfield `nano_banana_pro` + референс лица: автор в сцене + текст, editorial. Живо, лично.
- **Тип C · Скриншоты** — реальные скрины (таблица, Drive, результаты, переписка) + подписи. Build-in-public, доверие.
- **Тип D · Кейс-инфографика** — цифры, до/после, сравнения, диаграммы, графика.
За период **чередуй типы** (не 3 одинаковые). Через месяц по колонке «Зашло / Не зашло» анализируй, какой тип и тема заходят, и смещай план в их сторону.

## Шаг 2 — стратегия периода
По лестнице Ханта: баланс прогрев (1–3) / продажа (4–5) / вовлечение, привязка к событию.

## Шаг 3 — ПРЕДВАРИТЕЛЬНЫЙ план В ЧАТ (утверждение № 1 — НЕ в таблицу)
Покажи план прямо в чате — компактно, с ТИПОМ каждой карусели:
```
День/Дата · Формат (+тип A/B/C/D для карусели) · Тема · Идея/угол · Хук · Ступень Ханта · CTA
```
Без детальных промптов, без записи в таблицу. Каждая тема — сильная, конкретная. Спроси: «Утверждаешь план?» Итерируй до одобрения.

## Шаг 3.5 — ОБРАЗЕЦ из 2 постов (утверждение № 2, последнее)
Возьми из плана **2 поста РАЗНЫХ форматов (напр. 1 карусель + 1 reels)** и распиши ПОЛНОСТЬЮ, детально — как пойдёт в таблицу (все колонки + **промпт на каждый слайд**, см. Шаг 4). Покажи в чате. Спроси: **«Стиль и качество ок? Так делаю остальные?»** Это ЕДИНСТВЕННЫЙ контроль качества промптов.
Перед любым текстом: `Read ${CLAUDE_PLUGIN_ROOT}/skills/content-pipeline/references/copy-engine.md` + аудит (нет «не X, а Y», нет скобок-рода).

## Шаг 4 — ВСЯ ТАБЛИЦА + ПРОМПТ НА КАЖДЫЙ СЛАЙД (обязательно)
После одобрения образца — распиши ОСТАЛЬНЫЕ строки в ТОМ ЖЕ качестве и СРАЗУ запиши ВСЮ таблицу. Не спрашивай утверждение по каждому.
Лист «Контент-план», статус = «План». **ФИКСИРОВАННАЯ схема 14 колонок, БЕЗ эмодзи:**
```
Дата | Формат | Тема | Хуки (3 варианта) | Ступень Ханта | CTA / ключ | Higgsfield-промпт / Дизайн | Voiceover / Текст слайдов | Медиа (из банка) | Источник для скриншота | Ссылка на медиа (Drive) | Статус | Postiz (запланировано) | Зашло / Не зашло
```

### Колонка «Higgsfield-промпт / Дизайн» — ПРОМПТ НА КАЖДЫЙ СЛАЙД (критично)
Для каруселей эта колонка — **НЕ общий абзац, а ПРОНУМЕРОВАННЫЙ промпт на КАЖДЫЙ слайд:**
```
Слайд 01 (обложка): <модель/код> · <тип A/B/C/D> · сцена/фон · где аватар (референс лица) · какое фото из банка · палитра · текст-на-слайде reads-exactly · графика
Слайд 02: …
…
Слайд NN (финал): … + плашка с кодовым словом
```
- **Тип B** (фото/аватар): на обложке/финале — аватар в сцене (`nano_banana_pro` + medias reference лица), средние — рич-дизайн с графикой.
- **Тип A** (текст): код-вёрстка по `design-system` (HTML→Chrome).
- **Тип C** (скрины): что за скрин и какая подпись.
- **Тип D** (инфографика): что за цифра/график/сравнение.
Каждый слайд САМОДОСТАТОЧЕН: агент-генератор берёт строку и делает без домысливания. Для Reels — покадровый сценарий + промпт на каждый кадр.

**РАБОЧАЯ таблица, пополняется:** неделя → +неделя → месяц НА ТОМ ЖЕ листе.
Оформление — **никогда чёрный текст на белом:** шапка тёмный фирменный фон + белый жирный текст + закреплена + без эмодзи; тело бренд-беж + перенос по словам; ширина колонок без обрезки; Статус — выпадающий список (План · На утверждении · Утверждено · Запланировано · Опубликовано). Цвета — из `design-system`. Покажи ссылку.

## Шаг 5 — производство (по утверждению)
Для выбранных строк вызывай `content-pipeline`/формат-скилл по готовым послайдным промптам. Медиа → Drive, ссылка → в строку, статус → «На утверждении». QA визуала (кириллица/лицо/сцена) с перегенерацией брака.

## Шаг 6 — цикл утверждения и автопостинг
В таблице «Утверждено» / «Переделать». По «Утверждено» — автопостинг (статус → «Запланировано» → после даты «Опубликовано»). См. рутины `content-routine`.

## Автогенерация плана (рутина)
План может генерироваться сам: рутина «воскресенье → сгенерируй план на следующую неделю (или 2 недели / месяц) по этому скиллу, 3 карусели + 3 reels разных типов, статус План».

## Контроль качества
- Порядок: план в чат → образец 2 постов → (одобрение) → вся таблица автоматом. Только 2 утверждения.
- Каждая карусель имеет тип A/B/C/D, типы чередуются?
- У каждого слайда/кадра — свой детальный промпт (сцена/аватар/фото/стиль)?
- Микс 3 карусели + 3 reels на неделю?
- 14 колонок без эмодзи, статус-дропдаун, шапка закреплена, текст не обрезан?
- Текст без «не X, а Y» и скобок-рода?
