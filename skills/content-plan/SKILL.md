# Content Plan — оркестратор контент-плана

Назначение: превратить цель пользователя в **контент-календарь** и довести до готового контента. Это центральный скилл: он читает весь фундамент и дирижирует производством. На выходе — план на 1–4 недели + (по утверждению) готовые карусели/видео/посты в Google Drive.

## Когда срабатывает
Пользователь просит план контента на период или говорит «нужен контент на N недель, задача — X».

## Шаг 0 — собрать контекст (обязательно)
Прочитай профили из фундамента. Если какого-то нет — предложи сначала запустить нужный конструктор:
- `business-dna` — цель, продуктовая лестница, даты.
- `audience-builder` — аватары, боли, язык.
- `hunt-ladder` — уровни осознанности.
- `offer-builder` — оффер, гарантия, лестница.
- `tone-of-voice-builder` — голос.
- `design-system-builder` — палитра, шрифты, сетка, правила читаемости.
Без `business-dna` и `audience` план строить нельзя — попроси их собрать.

### Шаг 0.5 — дизайн-система ПЕРЕД планом (обязательно)
Если профиля `design-system` нет — **сначала запусти `design-system-builder`**, не строй план без неё. Прямо спроси человека: какие **цвета** ему нравятся (или покажи 2–3 готовые палитры на выбор) и какие **шрифты** (заголовочный + текстовый). Собери профиль `design-system` и только потом строй план. Таблица и весь визуал будут в этих цветах и шрифтах.

## Шаг 1 — зафиксировать цель и период
Уточни (если не сказано): цель в числах, срок, ключевое событие периода (вебинар/запуск), сколько публикаций в день/неделю реалистично.

## Шаг 2 — собрать стратегию периода
На основе лестницы Ханта распредели баланс: сколько прогрева (ступени 1–3), сколько продажи (4–5), сколько вовлечения/доверия. Привяжи к ключевому событию (за N дней до — прогрев, последние дни — продажа/дедлайн).

## Шаг 3 — построить календарь
Сгенерируй таблицу-календарь. Каждая строка:
```
Дата | Формат | Тема | Хук | Ступень Ханта | CTA | Статус | Зашло/Не зашло
```
Форматы бери из системы: reels, carousel, short-post, long-post, stories, article. Темы — из болей/желаний аудитории и продуктовой лестницы. Хуки — языком клиента (из hooks-database). Каждая единица работает на цель периода. Колонка «Зашло/Не зашло» пока пустая — её заполняют после публикации по метрикам (аналитика).

## Шаг 4 — сохранить план в КРАСИВУЮ таблицу
Запиши календарь в Google-таблицу (лист «Контент-план»). Колонка «Статус» = «План».
**Таблица создаётся ОФОРМЛЕННОЙ по дизайн-системе — НЕ чёрный текст на белом** (через GOOGLESHEETS_FORMAT_CELL + GOOGLESHEETS_UPDATE_SHEET_PROPERTIES):
- **Шапка (строка 1):** фон — фирменный тёмный цвет из `design-system` (напр. pine #00311e), текст белый, жирный, по центру; строка закреплена (freeze row 1).
- **Тело:** мягкий фон бренд-палитры (напр. beige #fef7e5), тёмный текст, перенос по словам, выравнивание сверху.
- **Ширина колонок** разумная (Тема/Хук/CTA — шире), авто-ресайз.
- **Статусы — цветом:** План — серый, На утверждении — жёлтый, Утверждено — зелёный, Опубликовано — pine/синий.
- **Колонка «Зашло/Не зашло»** — в том же стиле шапки и тела (не выпадает из оформления).
Все цвета берутся из профиля `design-system` пользователя. Затем покажи сводку и спроси, что генерировать.

## Шаг 5 — запустить производство (по утверждению)
Для выбранных строк вызывай `content-pipeline` (или конкретный формат-скилл). Перед генерацией каждого формата — прочитай его SKILL.md и единый движок текста:
```
Read: ${CLAUDE_PLUGIN_ROOT}/skills/content-pipeline/references/copy-engine.md
Read: ${CLAUDE_PLUGIN_ROOT}/skills/reels/SKILL.md
Read: ${CLAUDE_PLUGIN_ROOT}/skills/carousel/SKILL.md
... и т.д. по нужным форматам
```
Сгенерированный текст клади в строку. Картинки и видео загружай на Google Drive, ссылку вписывай в строку, статус → «Готово, на утверждение».

**Визуальные промпты — сразу идеальные (без ручной правки).** Перед любым Higgsfield-промптом прочитай стандарт:
```
Read: ${CLAUDE_PLUGIN_ROOT}/skills/carousel/references/higgsfield-visual-standard.md
```
Каждый слайд/кадр — своя сцена, реальные фото-референсы лица, единый образ, бренд-палитра, текст построчно «reads exactly: …» + «every word spelled correctly». **ОБЯЗАТЕЛЬНЫЙ QA** каждого результата (кириллица/лицо/сцена) с перегенерацией брака.

## Шаг 6 — цикл утверждения
Пользователь в таблице ставит «Утверждено» / «Переделать». По «Переделать» — перегенерируй с правкой. По «Утверждено» — готово к публикации (статус «К публикации»).

## Контроль качества
- Дизайн-система собрана ПЕРЕД планом и таблица оформлена в её цветах?
- Каждая единица плана привязана к цели периода и ступени Ханта?
- Баланс прогрев/продажа соответствует событию периода?
- Хуки — языком клиента, без «не X, а Y» и скобок-рода?
- План реалистичен по объёму?
- Медиа реально на Drive и ссылки в таблице?
- Визуал по `higgsfield-visual-standard.md` и прошёл QA?
