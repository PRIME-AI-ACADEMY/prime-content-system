# Content Pipeline — конвейер производства контента

Назначение: превратить один исходник в набор готовых форматов под профиль пользователя, сохраняя в Google Drive/Таблицах.

## Движок текста (ОБЯЗАТЕЛЬНО для всех текстов)
Весь текст любого формата пишется по единому движку **`references/copy-engine.md`** (анти-ИИ голос, банк хуков, вирусные паттерны, сценарные каркасы, CTA-движок на рост). Каждый форматный скилл тоже его читает.

## Шаг 0 — профили
Прочитай `audience`, `offer`, `tone-of-voice` (и `design-system` для визуала). Нет — собери через конструкторы.

## Основной процесс
1. **Золотые куски:** извлеки минимум 5 (тип · суть · точная цитата · подходящие форматы).
2. **Генерация:** по каждому формату вызови свой скилл (reels/carousel/stories/short-post/long-post/article).
3. **Сохранение:** в таблицу, одна единица контента на строку.
4. **Контроль качества:** прогони каждый текст через самопроверку движка (раздел 9): живой язык, конкретика, ноль канцелярита, CTA на рост.

## Критерии качества
- Преобразование материала, а не пересказ.
- Посты — короткие фразы (12–15 слов), reels — 8–10.
- Ноль канцеляризма и AI-штампов. Соответствие tone-of-voice.
- Каждая единица — с CTA на рост (сохрани/перешли/подпишись/слово в директ).

Режимы: полный · анализ-только · выборочный.
