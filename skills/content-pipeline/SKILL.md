# Content Pipeline — конвейер производства контента

Назначение: превратить один исходник в набор готовых форматов под профиль пользователя. Вызывается напрямую или из content-plan. Storage — Google Drive/Таблицы.

## Шаг 0 — профили
Прочитай offer, audience, tone-of-voice. Контент пишется под них. Нет профилей — предупреди и предложи собрать фундамент.

## Режимы
Полный (исходник → золотые куски → все форматы) / Только анализ / Выборочный.

## Шаги
1. **Золотые куски** (минимум 5): Тип (провокация/история/триггер/инсайт/механика/цитата), Суть, точная Цитата, почему зацепит, лучшие форматы. Точные цитаты, не пересказ.
2. **Генерация**: перед каждым форматом ОБЯЗАТЕЛЬНО прочитай его инструкцию:
```
Read: ${CLAUDE_PLUGIN_ROOT}/skills/reels/SKILL.md
Read: ${CLAUDE_PLUGIN_ROOT}/skills/carousel/SKILL.md
Read: ${CLAUDE_PLUGIN_ROOT}/skills/short-post/SKILL.md
Read: ${CLAUDE_PLUGIN_ROOT}/skills/long-post/SKILL.md
Read: ${CLAUDE_PLUGIN_ROOT}/skills/stories/SKILL.md
Read: ${CLAUDE_PLUGIN_ROOT}/skills/article/SKILL.md
```
Генерируй строго по инструкции, голосом из ToV.
3. **Сохранить**: каждая единица — строка/карточка в Google-таблице (лист «Контент»).
4. **Контроль качества (главное правило)**: ты не пересказываешь исходник — ты перерождаешь его. Звучит как живая речь. Цепляет с первых слов? Конкретика? Эмоция? CTA? Нет канцелярита («является»→«это» и т.д.). Фразы ≤ 12–15 слов (посты), 8–10 (reels). Голос сверь с профилем ToV.
