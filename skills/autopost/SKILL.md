# Autopost — автопостинг (3 режима)

Назначение: опубликовать/запланировать готовый контент из контент-плана. Выбери режим по тому, что подключено.

## Какой режим выбрать
- **Postiz подключён** → режим A (полный авто, много каналов; платно после триала).
- **Есть Meta Graph API токен** → режим B (полный авто в Instagram, бесплатно).
- **Ничего из этого** → режим C (Meta ручной-препост, бесплатно, полу-авто). По умолчанию для тех, кто не платит.

## Вход (любой режим)
Строка контент-плана: текст/подпись, ссылка на медиа (Google Drive), дата, статус. Отбирай строки: Статус = «Утверждено», есть медиа, отметка Postiz/Meta пуста. Не падай, если постить нечего.

## Режим A — Postiz
1. Composio GOOGLESHEETS_BATCH_GET → отбор строк.
2. Медиа из Drive → Postiz `uploadFromUrlTool`.
3. `integrationSchedulePostTool`: integrationId из `integrationList`, текст = подпись, дата из плана, type="schedule".
4. Статус → «Запланировано» + отметка Postiz.

## Режим B — Meta Graph API (бесплатный полный авто)
Требует разовой настройки (см. ниже). Instagram Content Publishing API:
1. `POST /{ig-user-id}/media` — image_url (прямая ссылка на медиа) + caption → получи creation_id (контейнер). Для карусели — контейнер на каждый слайд + родительский carousel-контейнер.
2. `POST /{ig-user-id}/media_publish` — creation_id → публикация.
   (Планирование на будущее — через отложенный вызов рутиной в нужную дату.)
3. Статус → «Опубликовано».
Вызовы — через Bash `curl` к graph.facebook.com с токеном из локального `.env` (не храни токен в таблице/репо). Медиа должно быть по ПУБЛИЧНОЙ ссылке (Drive — открыть доступ «по ссылке»).

### Разовая настройка Meta Graph API (для режима B)
1. Instagram — бизнес-аккаунт, привязан к Facebook-Странице.
2. developers.facebook.com → создай App (тип Business) → добавь Instagram Graph API.
3. Получи long-lived access token + ig-user-id (через Graph API Explorer, permissions: instagram_content_publish, pages_read_engagement).
4. Сохрани в локальный `.env` проекта: META_TOKEN, IG_USER_ID.
(Токен вводит сам пользователь — не проси его в чат; пусть положит в .env сам.)

## Режим C — Meta ручной-препост (бесплатно, полу-авто)
Система не постит, а готовит всё к вставке в Meta Business Suite:
1. Для каждой утверждённой строки собери ФИНАЛЬНУЮ подпись (хук + тело + CTA + 5 хэштегов) и убедись, что медиа в Drive открыто по ссылке.
2. Впиши в таблицу: статус «Готово к Meta», в колонку Postiz — готовую подпись и дату/время публикации.
3. Дай короткую сводку: «Готово N постов — открой business.facebook.com → Планировщик → вставь медиа и подпись».
Человек тратит ~2 мин/пост в планировщике Meta — бесплатно и надёжно.

## Правила
- Публикуй/готовь ТОЛЬКО строки со статусом «Утверждено».
- Подпись — голосом бренда (ToV), обращение на «ты», без «не X, а Y».
- Токены/секреты — только в локальном .env, НИКОГДА в таблице/репо/чате.
- Chrome-автоматизация Meta для ФОНОВОЙ рутины НЕ используется (Meta банит ботов).

Коннекторы: Composio (Google Sheets + Drive); Postiz (режим A); Bash+curl и Meta-токен (режим B).
