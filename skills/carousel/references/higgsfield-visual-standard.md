# Higgsfield Visual Standard (PRIME AI)

Единый стандарт визуала для всех скиллов, которые генерят изображения/видео с человеком в кадре через Higgsfield (carousel, reels, content-plan). Цель: промпты выходят **готовыми с первого раза** — интересные сцены, точное лицо, правильный русский текст, бренд-стиль. Никакой ручной правки каждой единицы.

Применяй этот стандарт ВСЕГДА, когда пишешь Higgsfield-промпт. Это не «совет», а чек-лист, который обязателен.

---

## 1. Рецепт генерации (технический, по умолчанию)
- **Модель:** `nano_banana_pro` — лучшая по кириллице и тексту в кадре + фотореализм. (через API/MCP списывает ~2 кредита/картинку; «безлимит» Nano Banana 2 — только в веб-приложении руками, не через автоматизацию.)
- **Формат:** `aspect_ratio: "4:5"`, `resolution: "2k"` для каруселей; `"9:16"` для вертикального видео/сторис.
- **Референс лица:** 1–2 РЕАЛЬНЫХ фронтальных фото человека (`medias` с `role:"image"`). **НЕ использовать AI-аватар/Soul-превью как референс** — лицо «уплывёт» в чужое. Реальные фото держат сходство.
- **Где взять фото:** если их нет как media_id — открой `media_upload_widget`, пусть человек выберет фото (браузер грузит напрямую; из песочницы байты залить нельзя).

## 2. Сцены — РАЗНЫЕ и интересные (главное правило)
- **Запрещено** повторять одну композицию (человек на тёмном фоне) из слайда в слайд. Каждый слайд = **своя сцена**: меняй окружение, масштаб (эпичный общий / кокпит / крупный / сверху / средний), ракурс, действие, реквизит.
- Под каждое сообщение — **конкретная метафора**, и весь набор слайдов складывается в **визуальную историю-арку**.
- **Один фирменный образ одежды** на всю карусель (для консистентности персонажа). Формула в каждом промпте: `same woman as in the reference photos (keep her EXACT face and identity), <signature outfit>`.
- **Ловушки буквального перевода — закрывай явно.** Модель понимает русские слова буквально:
  - «карусели» → `Instagram-style social-media carousel post slides (NOT amusement-park carousel rides / merry-go-rounds)`
  - «сторис/рилс» → `vertical Instagram story / reel UI on a phone`
  - «лента» → `social-media feed UI`, и т.п.

## 3. Текст на слайде (чтобы кириллица не плыла)
- Встраивай ТОЧНО, построчно: `Headline reads exactly: X. Second line reads exactly: Y. Third line reads exactly: Z.`
- В каждый промпт добавляй: `Clean legible Cyrillic, every word spelled correctly, bold beige.`
- **Длина:** заголовок + до 2 коротких строк (≤ ~6 слов в строке). Больше текста = выше риск искажения буквы. Развёрнутые описания уходят в подпись под пост, не в картинку.
- **Ключевое слово** (напр. `СИСТЕМА`) — отдельной строкой как подсвеченная плашка: `large highlighted keyword on its own: СИСТЕМА`.
- **Кавычки:** по умолчанию `No quotation marks` (ёлочки » « модель часто разворачивает наружу — »X«). Если нужны — проверяй направление на QA.
- Текст — в чистой/тёмной зоне (слева или снизу-слева) для читаемости. Не на лице, не в правых углах IG.

## 4. Бренд-палитра (дефолт; перекрывается `design-system` пользователя)
- Pine green `#00311e` + warm beige `#fef7e5`. В промпт: `Deep pine green / warm beige palette, cinematic, photorealistic, premium, not cartoon.`
- Свет — кинематографичный (god-rays / тёплый боковой / контровой), лёгкая дымка.

## 5. Анти-NSFW (чтобы не ловить ложный флаг и не жечь кредиты)
- Всегда указывай одежду и деловой/элегантный образ: `fully clothed in an elegant tailored suit`. Без этого портреты по референсу иногда падают в ложный NSFW-флаг (генерация впустую).

## 6. ОБЯЗАТЕЛЬНЫЙ контроль качества (перед выдачей)
Сгенерил → **посмотри каждый слайд** (открой картинку, сделай скриншот). Проверь:
1. **Текст:** кириллица верная, без лишних/потерянных букв, без развёрнутых кавычек.
2. **Лицо:** это тот человек (сверь с референсом).
3. **Сцена:** соответствует смыслу, нет буквальной ловушки (карусели-аттракционы и т.п.).
Любой слайд, который не прошёл, — **перегенерь** (≈2 кредита). Не отдавай брак пользователю на ручную ловлю.

## 7. Пайплайн доставки (без скачивания байтов)
- URL картинки Higgsfield → Google Drive папкой через `GOOGLEDRIVE_UPLOAD_FROM_URL` (серверный импорт; локально байты не тянем — сеть песочницы закрыта).
- Ссылку на папку/файл → в строку контент-плана (колонка «Ссылка на медиа»), статус → **«Готово — на утверждение»**.
- Google Drive должен быть подключён к ТОМУ аккаунту, где таблица и постинг (не перепутать аккаунты).

## 8. Видео / Reels (lip-sync) — специфика
- Лицо/сходство — те же реальные фото-референсы; один образ.
- Озвучка: клон голоса пользователя (если дал сэмпл) либо качественный русский голос; реплики короткие.
- Длинный скрипт режь на короткие сегменты (лимиты модели), собирай встык; субтитры/капшены — Submagic/CapCut.
- B-roll сцены — по правилам сцен выше (разные, кинематографичные).
