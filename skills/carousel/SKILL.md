# Carousel — Instagram-карусель (текст + дизайн кодом, merged)

Назначение: собрать карусель целиком — продающий текст слайдов + визуал, свёрстанный КОДОМ (не Canva) по дизайн-системе пользователя. Читает `audience`, `offer`, `tone-of-voice` и `design-system`. Метод dark-рендера адаптирован из скилла dark-carousel (автор Stanislav Tsys, @stastsis).

## Шаг 0 — контекст и бриф
Прочитай профили. Если не хватает — задай бриф одним сообщением: ниша/продукт · кто читает · одна главная боль · цель (охват-сохранения / доверие / продажа) · тон · есть ли цифры/кейсы.

## Чего нельзя
Водянистые вступления на первом слайде; «как мы все знаем…», «в современных реалиях…»; пассивный залог; больше 60 слов на слайд; абстракции вместо конкретики.

## Шаг 1 — выбрать тип карусели (5 типов)
1. **Список** — «[ЧИСЛО] вещей которые [результат]». Цель: охват, сохранения.
2. **Пошаговая инструкция** — «Как [результат] за [шаги]». Цель: доверие.
3. **До/после** — было → что изменила → стало. Цель: продажа, доверие.
4. **Мифы и правда** — МИФ → ПРАВДА (чередование). Цель: виральность, комментарии.
5. **История-кейс** — каждый слайд = поворот истории. Цель: вовлечение.

## Шаг 2 — текст слайдов
- **Слайд 1 — обложка** (самая важная, её видят в ленте): хук, который останавливает скролл. Макс 7–8 слов, только конкретика, вызывает вопрос «а как?». Форматы: цифра+обещание / вопрос-триггер / провокация КАПСОМ / до-после / FOMO.
- **Слайд 2 — приземление**: сразу СУТЬ карусели одной фразой крупно (НЕ «о чём этот пост», НЕ вопрос). Мелким снизу — обещание структуры («↓ 6 кейсов»).
- **Слайды 3–N — раскрытие**: одна мысль на слайд, 5–8 слов в строке, активные глаголы, «ты». Конкретные шаги (не «пиши лучше», а «первое предложение из 5 слов»).
- **Доказательство** (предпоследние): кейс/цифра/результат — без него карусель слабее вдвое.
- **Финал — CTA**: одно действие («сохрани», «напиши слово [X] в директ», «отметь кому нужно»).
- **Текст под постом**: 2–3 предложения + CTA. Первое слово — НЕ «Я». Хэштеги — по желанию пользователя.

Объём: обложка ≤8 слов; обычный слайд ≤40–60 слов; финал ≤20 слов + CTA. Голос — из профиля ToV.

## Шаг 3 — дизайн кодом (принцип «холодного читателя»)
Пиши для человека, который НЕ знает темы до того как начал листать: каждый слайд самодостаточен, все цифры/термины подписаны. Жёсткие правила вёрстки (из dark-carousel):
- Слайд 1 = визуал/фото или генерация; запретные зоны IG — правый верхний и правый нижний угол (там текста нет).
- Ник пользователя — нижний левый угол каждого слайда (из `design-system`); нумерация `NN/TT` — верхний левый.
- Максимум 2 цвета текста на слайде; line-height ≥1.0 для крупных русских заголовков (Й/Ё/Щ не наезжают).
- Минимальные шрифты: footer/CTA ≥20px, body ≥22px. Не лезет — дроби на +1 слайд (лимит IG = 20), не сжимай.
Рендер: HTML/CSS → PNG 1080×1350 (по желанию MP4 с видео-фоном на 2–4 слайдах, текст светлый + затемнение). Стиль, цвета, шрифты — строго из `design-system` пользователя. Если её ещё нет — дефолтный editorial (тёмный фон, крупная типографика, один акцент) + предложи собрать дизайн-систему.
Экспорт: файлы на Google Drive, ссылка в строку контент-плана со статусом «на утверждение». Перед рендером — согласуй структуру (хуки + таблица текста) с пользователем.

## Шаг 3b — визуал через Higgsfield (по умолчанию, если человек в кадре)
Если карусель строится на фото/сценах с человеком (как референсы notionlabs / theaiagents), а не на чистой код-вёрстке — генерируй слайды через Higgsfield по стандарту **`references/higgsfield-visual-standard.md`** (ОБЯЗАТЕЛЬНО прочитай его перед написанием промптов). Коротко:
- `nano_banana_pro`, 4:5, 2k, 1–2 РЕАЛЬНЫХ фото человека как референс (не Soul-превью).
- Каждый слайд — СВОЯ интересная сцена (меняй окружение/масштаб/ракурс/действие), единый образ одежды, бренд-палитра pine/beige.
- Текст встраивай построчно «reads exactly: …» + «every word spelled correctly»; заголовок + ≤2 коротких строки; ключевое слово плашкой; ловушки буквального перевода («карусели» → соцсети-карусели, не аттракцион) закрывай явно; деловой образ — против ложного NSFW.
- **ОБЯЗАТЕЛЬНЫЙ QA:** посмотри каждый слайд глазами (скриншот) → проверь кириллицу, лицо, сцену → перегенери брак. Не отдавай на ручную правку пользователю.
- Доставка: HF-URL → Google Drive (`UPLOAD_FROM_URL`) → ссылка + статус «Готово — на утверждение» в строку плана.

## Шаг 4 — выдача текста
```
ОБЛОЖКА: [текст]
СЛАЙД 2: [суть] ...
ФИНАЛЬНЫЙ СЛАЙД: [CTA]
+ рекомендация по визуалу · подпись под пост (2–3 предл + CTA)
```

## Контроль качества
Обложка останавливает скролл за 1 сек? · слайд 2 = ответ, не вопрос? · один слайд = одна мысль, читается с телефона за 3 сек? · есть доказательство и нарастание к инсайту? · чёткий CTA? · дизайн единый из design-system, не «сток»? · голос из ToV, без канцелярита?
